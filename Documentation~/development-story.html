The Challenge

Traditional ScriptableObject event systems in Unity have always presented a dilemma: great for editor workflows but difficult to use programmatically. The lack of async support further limited their usefulness in modern development scenarios.

Limitations

  • Poor code-level access
  • No async support
  • Limited type safety
  • No progress tracking

Solution

  • Unified code/editor API
  • First-class async/await
  • Complete type safety
  • Progress tracking & cancellation

This event system finally bridges both worlds: the convenience of SO events with the power of code-based events, plus proper async support.

AI-Assisted Development

Built with Claude's assistance, this system was developed through a collaborative process where AI handled implementation details while I focused on architectural decisions and Unity-specific requirements.

We used a test-driven approach to ensure robustness across event registration, type constraints, async handling, cancellation, and ScriptableObject integration.

Real-World Use

The system excels in scenarios like:

  • Game State Management with progress reporting
  • Cloud Integration via async events with tracking
  • Editor Tools that work seamlessly in both contexts

It "just works" - whether from code or inspector, synchronously or asynchronously.
